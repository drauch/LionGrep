# Changelog

All notable changes to LionGrep are documented in this file.

The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and the project
adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Each release branch (`release/x.y.z`) requires a matching `## [x.y.z] - YYYY-MM-DD` section
below — the CI workflow reads this file to populate the GitHub Release body, and fails if it
can't find a matching entry. Sections are conventionally one or more of: **Added**, **Changed**,
**Fixed**, **Removed**, **Deprecated**, **Security**.
